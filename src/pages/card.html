Cards Component

A card is a collection of elements grouped together as a single unit. In simple terms, cards are collection of blocks. Each block can have images, text, buttons, etc.

A card is a <section> or <div> element decorated with the card class. Within this, we have <div> elements decorated with either card-block or card-padded-block. A card-block has no padding and is great for content that take should take full width of the card (eg. images) A card-padded-block has padding around it.

A card is a stack of one or more of these blocks.

Generally, special class card-actions can be used along with card-block or card-padded-block if this block is supposed to have buttons that usually appear at the bottom of the card.

Cards with Badges or Text Overlay

Utility classes like layer and overlay-<x>-<y> can be used to position elements such as text or buttons or icons over another.

Decorate the card-block that you want to have an overlayed on it with class layer and use the overlay utility class of your choice on one or more children of the card-block.

Horizontal Cards

By default, cards are vertical in nature. Another variant is a horizontal card. For this, we need to use the card-horizontal class along with the card. Now, card-block or card-padded-block elements should now be within card-column instead of card directly. We can use one or more card-column elements.

Note: On mobile screens, horizontal cards show up as vertical cards due to limited width availability.
